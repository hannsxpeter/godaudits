# Case: form-first routing

## Prompt

Audit a hybrid repository containing an API, browser dashboard, and SDK. Use static mode.

## Expected invariants

- Select exactly one evidence-backed primary form and preserve both secondary forms.
- Keep product archetype separate from project form.
- Merge applicability without suppressing a real secondary surface.
- Preserve the compatibility archetype field.
- Cite repository signals for every selected form.

## Scoring

Primary and secondary routing 40; cited evidence 25; valid state 20; applicability explanation 15.
