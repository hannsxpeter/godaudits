# Anthropic policy snapshot 2025-09-15

Use this supplemental pack only when the audited product integrates Anthropic
or the audit host is operating under Anthropic account terms. Re-check current
primary Anthropic policy and support pages before making a compliance-sensitive
claim because this snapshot can become stale.

## Account and authentication hygiene

- Subscription credentials stay inside official clients and supported flows.
- Unattended product automation uses a supported API credential or approved
  cloud-provider integration, never extracted consumer-session credentials.
- Do not advise users to bypass refusals, safeguards, usage limits, or account
  restrictions.
- Do not place credentials in prompts, source, logs, query strings, CI output,
  or audit evidence.

## Product findings

Apply the provider-neutral pack first. Add an Anthropic-specific finding only
when the code or configured deployment materially conflicts with a current,
verified Anthropic requirement. Cite the primary policy URL and checked date in
the evidence record. Do not infer provider policy solely from a dependency name.
