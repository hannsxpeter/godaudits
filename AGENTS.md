# AGENTS.md

This repository ships the **godaudits** Agent Skill: an audit superskill that
inspects an existing codebase end to end and emits one master audit report
(`.godaudits/AUDIT.mdx`) with per-domain scores, evidence-backed findings,
and an agent-executable remediation plan.

If you are an agent asked to audit a project: read
`skills/godaudits/SKILL.md` and follow its method. The skill is also installed
via `.agents/skills/godaudits` and `.claude/skills/godaudits` projections in
consuming projects.

If you are an agent working on this repository itself:

- The canonical skill lives at `skills/godaudits/`; the `.agents/` and
  `.claude/` entries are symlinks to it. Edit the canonical files only.
- `PROMPT.md` is generated by `scripts/build-prompt.sh`. Never edit it by hand;
  regenerate it after changing SKILL.md or the inlined references.
- Style is mechanically enforced by `scripts/lint.sh` (run `npm run lint` or
  `bash scripts/lint.sh --all`): ASCII punctuation only, no em or en dashes,
  no Unicode arrows (write ASCII `->`), no emojis, no smart quotes, no
  box-drawing characters, in every authored file.
- Reference modules under `skills/godaudits/references/` follow a fixed
  six-section contract (Lineage, Surface map, Checks, Scoring, Remediation
  seeds, Anti-patterns hunted); the linter checks it.
- Check ids `A-<DOM>-n` mirror the godplans requirement ids `R-<DOM>-n` one
  to one; do not renumber one side without the other.
- Version appears in three places that must agree: SKILL.md frontmatter
  `metadata.version`, the SKILL.md body `## Skill version:` line, and the top
  CHANGELOG.md entry. The linter checks that too.
